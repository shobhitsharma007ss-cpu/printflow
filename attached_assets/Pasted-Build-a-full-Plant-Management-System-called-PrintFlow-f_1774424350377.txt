Build a full Plant Management System called "PrintFlow" for a printing and packaging unit. This is a real production system — it must be easy to use for non-tech factory workers.

TECH STACK:
- Node.js + Express backend
- SQLite with better-sqlite3
- React frontend (Vite)
- Recharts for data visualization
- React Router for navigation
- All in one Replit project

---

DATABASE SCHEMA:

1. vendors — id, vendor_name, contact_person, phone, city, created_at

2. materials — id, material_name, material_type (board/paper/consumable), 
   sub_type (grey-back/white-back/fbb/sbs/art-card/maplitho/uv-ink/led-uv-ink/
   varnish/aqueous-coating/cmyk-ink/gum/lubricant/other),
   gsm (nullable), unit (reams/kg/litre/sheets), 
   current_qty, min_reorder_qty, created_at

3. material_vendors — id, material_id (FK), vendor_id (FK) — many to many

4. stock_inward — id, material_id (FK), vendor_id (FK), qty_received, 
   unit, batch_ref, received_date, notes

5. machines — id, machine_name, machine_code, machine_type 
   (printing/cutting/coating/gluing), max_paper_width, max_paper_length,
   speed_per_hour, capabilities (JSON array: uv/varnish/non-woven),
   status (idle/running/maintenance), operator_name, notes

6. job_templates — id, template_name, description, 
   routing_steps (JSON array of machine_ids in order)

7. jobs — id, job_code (PF-001 auto), job_name, client_name, 
   material_id (FK), material_gsm, qty_sheets, 
   planned_sheets, status (pending/in-progress/completed/on-hold),
   template_id (FK nullable), scheduled_date, created_at

8. job_routing — id, job_id (FK), step_number, machine_id (FK), 
   operator_name, status (pending/in-progress/completed), 
   started_at, completed_at, notes

9. job_materials — id, job_id (FK), material_id (FK), 
   planned_qty, actual_qty, unit, cost_per_unit

10. wastage_log — id, job_id (FK), material_id (FK), 
    planned_qty, actual_qty, wastage_qty, wastage_pct,
    reason (setup/mis-registration/client-correction/plate-change/other), logged_at

FRONTEND — VERY IMPORTANT — NON-TECH USERS, MUST BE VISUAL:

Use React Router. Layout: fixed left sidebar (dark, #1a1a2e), main content area (light grey bg).

Sidebar links with icons:
- Dashboard (home)
- Floor Monitor (machines)
- Jobs
- Inventory
- Reports
- Settings

PAGE 1 — DASHBOARD:
- Top: 4 metric cards (Active Jobs, Machines Running, Low Stock Alerts, Jobs Completed Today)
- Middle: Machine status row — each machine as a card with its name, operator, current job name or "IDLE", colored indicator (green=running, grey=idle, red=maintenance)
- Bottom: 3 recent jobs with status badges

PAGE 2 — FLOOR MONITOR:
Full screen layout. Each machine as a large card with:
- Machine name big and bold
- Type badge (PRINTING / CUTTING / COATING / GLUING)
- Operator name
- Status indicator (big colored dot — green/grey/red)
- Current job name or "IDLE"
- Speed spec
Group machines by type: Printing Machines, Cutting, Coating, Gluing

PAGE 3 — INVENTORY (MOST VISUAL PAGE):
Split into two tabs: Boards & Paper | Consumables

For Boards & Paper:
Show each material as a VISUAL STACK — a rectangular block where the HEIGHT of the block represents quantity relative to max stock. 
- Full stack = green
- Half stack = amber  
- Low stack = red with warning
Show material name, GSM, current qty, unit below each stack.
Clicking a stack opens a side panel with vendor info and inward history.

For Consumables:
Show as gauge/cylinder visuals — a vertical cylinder filled proportionally.
Same color logic (green/amber/red).

PAGE 4 — JOBS:
Table of all jobs with columns: Job Code, Job Name, Client, Material, Qty, Status badge, Assigned Machine(s), Scheduled Date, Action buttons.
Filter by status. 
"New Job" button opens a form:
- Job name, client name
- Select material + GSM
- Select template (auto-fills routing) OR build custom routing
- Enter qty sheets
- Schedule date
On save: auto-generates PF-XXX code, creates job_routing steps, deducts planned material from stock.

PAGE 5 — REPORTS:
Three report cards:
- Wastage Report: table per job showing planned vs actual with wastage % bar
- Stock Report: all materials with current qty, reorder status
- Job Cost Report: per job total material cost breakdown

- Sidebar: dark navy #1a1a2e with white text

- Status colors: green #22c55e, amber #f59e0b, red #ef4444, blue #3b82f6
- All tables have alternating row colors

- Buttons are large and clearly labeled
